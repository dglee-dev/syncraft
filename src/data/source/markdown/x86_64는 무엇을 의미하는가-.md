# x86_64는 무엇을 의미하는가?

네오빔(Neovim)을 설정하기 위해 여러가지 패키지를 설치하던 중 CPU에 따라 ARM64와 x86_64용 패키지를 구분해서 설치해야 함을 발견했다.

<img src="https://justin-cms-images.s3.ap-northeast-2.amazonaws.com/86_64_3554483f07.png">

ARM64가 최신의 M1 등 맥 실리콘을 탑재한 컴퓨터, x86_64가 인텔 CPU 기반 컴퓨터를 의미한다는 것을 알고는 있지만 정확히 이 숫자들이 무엇을 의미하는 것인지 궁금해져서 찾아보았다.

먼저 x86에 대한 위키피디아의 글을 읽어보자.

> x86, also known as `80x86` or the `8086 family` -  is a family of complex instruction set computer (CISC) instruction set architectures initially developed by Intel based on the Intel 8086 microprocessor and its 8088 variant.

x86은 기본적으로 instruction set 'architecture', 즉 설계 구조를 의미하는 것이고, 이것은 인텔의 8086 마이크로 프로세서를 기반으로 만들어진 것이다.

> The term 'x86' came into being because the names of several successors to Intel's 8086 processor end in '86', including the 80168, 80286, 80386 and 80486 processors.

어릴 때 들었던 268 368 468 컴퓨터 뭐 그런 용어들과 무관하지 않다는 말이다. 이 프로세서들의 번호 뒷자리가 '86' 으로 끝나니까 'x86' 으로 통칭한 것.

그럼 `x86_64` 의 `64` 는 무엇을 의미할까? 

결론부터 말하자면, 64는 '64비트' 를 의미하고, 이 비트 수는 컴퓨터의 프로세서(CPU)가 정보를 처리하는 processing capability - 메모리 가용량을 의미한다. 역사적으로 컴퓨터는 2bit, 4bit, ...32bit를 거쳐 현재의 64bit 컴퓨팅까지 진화 해 왔다. 잠시 예를 들어보면, 컴퓨터 게임을 만든다고 할 때 8비트 컴퓨터로는 2^8 = 256가지의 색만 표현할 수 있고, 64비트 컴퓨터로는 2^64 = 1천 8백 경(..) 가지의 색을 표현할 수 있으므로 하늘과 땅 차이라 해도 과언이 아니다.

사진은 1991년 '마이크로소프트웨어' 잡지에 실린 것. 16비트였던 286 컴퓨터는 메모리 가용량이 16'MB' 였던 것에 비해, x86 아키텍처를 채택하여 32비트의 시작을 알리는 386 컴퓨터에서는 4'GB' 의 메모리 가용량을 자랑했다.

<img src="https://justin-cms-images.s3.ap-northeast-2.amazonaws.com/386_ac96ccad45.jpeg">

이후 인텔이 2001년 아이태니움(Itanium)이라는 이름으로 64비트 CPU를 발표했고 잇따라 AMD는 2003년 옵테론(Opteron)을 발표, x86-64 ISA를 탑재해 32비트 환경과의 호환성을 힘입어 인텔을 앞서기 시작했고, 결국 인텔도 AMD가 개발한 x86-64 ISA(Instruction Set Architecture)를 차용하게 된다.

따라서 x86-64는 64비트 컴퓨팅이 가능하면서도 32비트를 호환하는 아키텍처를 의미한다. 이는 64비트 컴퓨팅으로 해석해도 무방하기에 현대에는 줄여서 `x64` 로 간단하게 표기할수도 있다. 반대로 `x86` 은 32비트에서 사용되던 아키텍처 네이밍이고, 따라서 이는 32비트 컴퓨팅을 지원하는 아키텍처, 혹은 CPU, 혹은 해당 CPU를 사용하고 있는 컴퓨터를 의미한다.

그런데 instruction set? CISC는 뭐야? 궁금하니까 조금 더 알아보자.

## 1. Instruction Set

> Instruction set is a collection of instructions that a processor can execute.

 프로세서가 실행할 수 있는 명령의 한 집합을 말한다고 한다.

> Each instruction typically represents a basic operation that the processor can perform, such as arithmetic operations(add, sub, multiple, division), logical operations (AND, OR, NOT), data movement operations (loading data from memory, storing data to memery), and control flow operations(branching, jumping).

각각의 instruction이 어떤 형태의 활동(일반적으로 `연산` 이라고 칭하는 것 같다)을 수행할 수 있는지 말해준다.

> The instructions in an instruction set are encoded as binary patterns, and the processor's instruction decoder interprets these patterns to carry out the corresponding opertions.

instruction set에 있는 instruction들은, 바이너리 패턴으로 인코딩 되어있으며 프로세서의 instruction decoder가 이것을 해독해 각각에 걸맞는 operation을 수행할 수 있게 된다.

> Instruction sets vary between different processor architectures. For example, the x86 instruction set used in Intel and AMD processors is different(!) from the ARM instruction set used in many mobile devices..(중략)

프로세서 아키텍처에 따라 instruction set은 다르며, 따라서 개발자들은 아키텍처의 종류에 주의해야 한다.

## 2. RISC and CISC

먼저 공통적인 `~ISC` 는 `Instruction Set Computer` 의 약자이다. 컴퓨터는 CPU를 의미하고, Instruction Set은 CPU의 작업 처리를 정의한 명령체계를 의미한다.

각각 처음에 붙는 알파벳인 R은 Reduced, C는 Complex를 의미한다. 따라서 'Reduced Instruction Set' Computer는 하나의 instruction에서 하나의 operation을 수행하는데 집중하고, 'Complex Instruction Set' Computer는 하나의 instruction에 여러개의 operation을 포함하는 구조의 CPU를 의미한다.

애플 실리콘(M1, M2, M3..)은 RISC를 기반으로 개발된 ARM의 CPU 아키텍처를 기반으로 개발되었고, 인텔의 x86등은 모두 CISC이다. 처음부터 ARM은 저전력으로 고효율을 낼 수 있는 RISC기반의 칩을 개발해왔고 인텔은 고성능을 위해 CISC기반의 CPU를 만들어왔지만 최근 애플이 ARM기반으로 애플실리콘을 만들면서 RISC 기반의 고성능 CPU들이 만들어져 애플의 기기들에 탑재되고 있다.

> 1980년대 중반, 영국의 컴퓨터 회사 Acorn Computer는 비즈니스용 고성능 컴퓨터를 만든다는 목표를 세운다. 하지만 새 컴퓨터에 쓸만한 기성품 CPU를 찾을 수 없자 직접 RISC 기반 CPU 아키텍처를 개발하기로 하는데, 이것이 `Acorn RISC Machine` 즉 ARM의 시작이다...

> ...1980년대 말, Apple은 Newton MessagePad에 들어갈 적절한 성능에 적절한 전력을 소모하는 마이크로프로세서가 필요했는데, 여기에 ARM 아키텍처를 쓰기로 결정한다. 이에 Acorn은 아예 ARM 개발 부문을 떼어내 Apple과 VLSI Technology와의 합작회사를 설립하고 이름을 ARM(Advanced RISC Machine)으로 변경했으며, 이후로는 주로 저전력 위주의 SoC 형태의 칩 개발에 집중하게 된다.
