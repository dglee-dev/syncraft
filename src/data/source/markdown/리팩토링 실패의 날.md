# 리팩토링 실패의 날

어제는 ACC 프로젝트 개발 중 주말에 시도한 리팩토링이 실패했다.
 
이를 반성하며..

1. 너무 많은 작업을 한번에 하려고 했다
  - 작은 성공(커밋)들로 쪼개자

2. 너무 많은 일을 하나의 모듈에서 해결하려고 했다
  - 유연성이 떨어지고 복잡도가 올라가서 예상대로 작동하지 않았을 때 혼란스러웠다
  - 만약 1번을 잘 했다면 어디에서 문제가 발생했는지 예측하기 편했을 것이다.

3. 충분한 테스트가 필요하다
  - 이렇게 저렇게 하면 되겠지? 하고 너무 많은 리팩토링 후 테스트 했을 때 예상치 못한 결과가 나왔다. 중간중간 테스트가 필요하다.
  - 1번과 비슷한 이야기로 귀결된다.
