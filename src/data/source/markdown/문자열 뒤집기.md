# 문자열 뒤집기

'주어진 문자열을 in-place 로 뒤집으시오' 라는 문제이다. 바로 풀이를 적어본다.

```js
function reverseString (str) {
  let start = 0;
  let end = str.length - 1;

  while (start < end) {
    swap(start, end, str);
  }
  
  return str;
}

function swap (start, end, str) {
  const temp = str[start];
  
  str[start] = start[end];
  str[end] = temp;
  
  return str;
}
```

내가 정리하고 싶었던 오늘의 질문은 다음과 같다.

> swap의 구현은 항상 temp 변수를 사용할 수 밖에 없다. temp는 새로운 공간인데 이를 in-place라고 할 수 있는가?

먼저 위키피디아에서 in-place를 정확히 무엇이라고 정의하는지 확인해보자.

> In computer science, an in-place algorithm is an algorithm that operates directly on the input data structure without requiring extra space **proportional to the input size.** In other words, it modifies the input in place, **without creating a separate copy of the data structure**.

requiring extra space 'proportioanl to the input size' 라는 말에 집중해보면, 인풋의 크기에 비례해서 새로운 공간을 필요로 하는 알고리즘을 의미한다는 것이다. 위의 알고리즘에서는 인풋의 크기가 늘어나면 그에 비례해 '공간을 사용하는 횟수' 는 늘어나지만 계속해서 필요로 하는 '공간의 크기' 는 문자 1개, O(1)이라고 말할 수 있다.

만약 인풋을 복사하는 새로운 결과물을 만들어내는 방식이라면 인풋의 크기에 비례하는 '공간의 크기' 가 필요할 것이다.

결과적으로, 어떤 알고리즘이 in-place인지 아닌지를 따져보고 싶다면(..애초에 따지지 말고 GPT에게 물어보자) 인풋의 크기에 비례하는 '공간의 크기' 를 새롭게 필요로 하는지에 대해 생각해보자.
