# S3 연결 흐름 설계

## 앱 목적

왼쪽 패널(S3 파일 목록) ↔ 오른쪽 패널(로컬 파일) 간 동기화 도구

---

## '연결' 버튼 클릭 시 흐름

1. **자격증명 암호화 저장** — AES-256-GCM으로 암호화 후 PostgreSQL에 upsert
2. **S3 API 호출** — 저장된 자격증명으로 `ListObjectsV2` 실행, 해당 버킷/prefix의 파일 목록 조회
3. **SourceDisplay 업데이트** — 헤더 텍스트를 입력한 버킷명 + 리전으로 교체
4. **SourceList 교체** — 로컬 파일 목록 대신 S3에서 가져온 파일 목록으로 대체
5. **폼 닫기**

---

## 앱 최초 로드 시 흐름

1. DB에서 저장된 자격증명 조회 및 복호화
2. 자격증명 있으면 → S3 파일 목록 자동 로드
3. 없으면 → 빈 목록 또는 연결 안내

---

## 구현 방향

**Server Action** 방식 채택.

- 자격증명이 서버에서만 처리돼 브라우저에 노출되지 않음
- Next.js 15와 잘 맞음
- 자격증명은 AES-256-GCM으로 암호화, 암호화 키는 서버 환경변수(`ENCRYPTION_KEY`)로 관리

---

## 상태 관리

- `s3ConnectionAtom` — 연결된 bucket, region, prefix 저장
- `s3FileListAtom` — S3에서 가져온 파일 목록 저장
- 두 atom 모두 Jotai 사용
